<objective>
Produce a two-stage Alpine `Dockerfile` (golang:1.22-alpine builder ŌåÆ alpine:3.19 runtime
with `ca-certificates`/`tzdata` and a non-root `botuser`), a `docker-compose.yml` that
loads `.env` and bind-mounts `./data` to `/app/data`, and a `.env.example` documenting
every required environment variable. `docker build .` must succeed and `docker compose
up -d` must start the bot.
</objective>

<must_haves>
  <truths>
    <truth>Dockerfile builder stage: `FROM golang:1.22-alpine AS builder`, copies `go.mod`/`go.sum`, runs `go mod download`, copies source, builds with `CGO_ENABLED=0 GOOS=linux go build -o /out/bot ./cmd/bot`.</truth>
    <truth>Dockerfile runtime stage: `FROM alpine:3.19`, `RUN apk add --no-cache ca-certificates tzdata`, creates non-root user `botuser` (uid/gid 1000), copies binary to `/app/bot`, sets `WORKDIR /app`, `USER botuser`, `ENTRYPOINT ["/app/bot"]`.</truth>
    <truth>docker-compose.yml defines a single `bot` service that `build: .`, sets `env_file: .env`, mounts `./data:/app/data`, and uses `restart: unless-stopped`.</truth>
    <truth>.env.example documents `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`, `DATA_DIR=/app/data` (the in-container path) ŌĆö no real secrets, only placeholders.</truth>
    <truth>.dockerignore excludes `.planning/`, `.claude/`, `.git/`, `*.md`, `data/`, `.env`, test artifacts ŌĆö keeps build context small and prevents leaking secrets into the image.</truth>
    <truth>`docker build .` exits 0 against the current working tree.</truth>
  </truths>
</must_haves>

<task id="05-01-T01">
  <title>Multi-stage Dockerfile</title>
  <read_first>
    - go.mod
    - cmd/bot/main.go
    - .planning/phases/05-docker-integration-e2e-tests/05-CONTEXT.md
    - original-prd.md (┬¦6 Dockerfile spec)
  </read_first>
  <action>
    Create `Dockerfile` at repo root with two stages:

    Builder (`builder` alias, `golang:1.22-alpine`):
    - `WORKDIR /src`
    - Copy `go.mod` and `go.sum`, run `go mod download`
    - Copy the rest of the source
    - Build: `CGO_ENABLED=0 GOOS=linux go build -ldflags="-s -w" -o /out/bot ./cmd/bot`

    Runtime (`alpine:3.19`):
    - `RUN apk add --no-cache ca-certificates tzdata`
    - Create non-root user: `RUN addgroup -S botuser && adduser -S -G botuser -u 1000 botuser`
    - `WORKDIR /app`
    - `COPY --from=builder /out/bot /app/bot`
    - `RUN mkdir -p /app/data && chown -R botuser:botuser /app`
    - `USER botuser`
    - `ENV DATA_DIR=/app/data`
    - `ENTRYPOINT ["/app/bot"]`

    Keep image minimal ŌĆö no shell wrappers, no scripts.
  </action>
  <acceptance_criteria>
    - `Dockerfile` exists at repo root with two FROM lines: `golang:1.22-alpine AS builder` and `alpine:3.19`
    - Runtime stage contains `apk add --no-cache ca-certificates tzdata`
    - Runtime stage creates and switches to `botuser` (USER botuser)
    - Builder uses `CGO_ENABLED=0 GOOS=linux go build -o /out/bot ./cmd/bot`
    - `docker build -t questionnairebot:test .` exits 0
    - `docker run --rm questionnairebot:test /app/bot 2>&1 | head -1` prints `FATAL: TELEGRAM_BOT_TOKEN is required` (proves binary runs as botuser and exits cleanly on missing config)
  </acceptance_criteria>
</task>
