<objective>
Implement `internal/scheduler` that registers one `cron/v3` job per loaded questionnaire,
each in its own IANA timezone via `time.LoadLocation`, logs the next-trigger time in RFC3339
for each questionnaire, and wires a stubbed callback (the real handler lands in Phase 4).
Wire the scheduler into `cmd/bot/main.go` so `go build ./... && go vet ./...` is clean.
</objective>

<must_haves>
  <truths>
    <truth>One *cron.Cron instance per questionnaire — each constructed with `cron.WithLocation(q.Location)` so the schedule fires in the configured timezone.</truth>
    <truth>The next-trigger log line is RFC3339 (with timezone offset). Format: `Next trigger for <slug>: <RFC3339>`. One line per questionnaire.</truth>
    <truth>Cron handlers are stubs in this phase — they log `cron fire (stub): <slug>` and return. No Telegram or session I/O.</truth>
    <truth>The scheduler exposes Start(ctx) and Stop() so main.go can shut it down on signal.</truth>
    <truth>Registration errors (e.g. AddFunc returning an error) abort startup with `FATAL: scheduler: <slug>: <err>`.</truth>
  </truths>
</must_haves>

<task id="01-03-T01">
  <title>Implement internal/scheduler.Scheduler</title>
  <read_first>
    - internal/loader/types.go
    - internal/loader/loader.go
    - .planning/REQUIREMENTS.md (SCHED-01, SCHED-02)
    - .planning/phases/01-foundation-config-loader-scheduler-init/01-CONTEXT.md
  </read_first>
  <action>
    Create `internal/scheduler/scheduler.go` exposing:
    - `type Handler func(slug string)` — the per-fire callback signature.
    - `type Scheduler struct { crons []*cron.Cron; slugs []string }`
    - `func New(qs []*loader.Questionnaire, h Handler) (*Scheduler, error)` —
        For each `q`:
        1. Construct `c := cron.New(cron.WithLocation(q.Location))`.
        2. Call `c.AddFunc(q.Schedule, func(){ h(q.Slug) })`. Use a local-variable
           capture (loop-variable safety on Go 1.22 is fine, but copy for clarity).
        3. Append c to `s.crons`, q.Slug to `s.slugs`. Return wrapped error on AddFunc failure:
           `fmt.Errorf("scheduler: %s: %w", q.Slug, err)`.
    - `func (s *Scheduler) Start(ctx context.Context)`:
        For each (cron, slug): call `c.Start()`, then read the registered entry's
        `Next` field via `c.Entries()[0].Next` and log
        `Next trigger for <slug>: <RFC3339>` to stdout.
        Launch a goroutine that calls `<-ctx.Done(); s.Stop()` so cancellation tears down all crons.
    - `func (s *Scheduler) Stop()`:
        For each cron, call `c.Stop()` (which returns a context but we don't need to wait here).
    The stub Handler is provided by main.go — in this phase it logs "cron fire (stub): <slug>".
  </action>
  <acceptance_criteria>
    - File `internal/scheduler/scheduler.go` exists
    - Exports `New(qs, handler) (*Scheduler, error)`, `Start(ctx)`, `Stop()`, and `Handler` type
    - On Start, stdout contains exactly one `Next trigger for <slug>: <RFC3339>` line per questionnaire
    - The RFC3339 timestamp logged matches the questionnaire's timezone offset (e.g. `+05:30` for Asia/Kolkata)
    - Cancelling the parent context causes all internal crons to Stop within 1 second
    - `go build ./internal/scheduler/...` and `go vet ./internal/scheduler/...` exit 0
  </acceptance_criteria>
</task>
