<objective>
Implement `commands.CronBus` which receives slug-fire events from `internal/scheduler`,
groups them within a 1-second window, and dispatches: silent no-op if cycle already
completed, auto-start if single fire and no session active, picker if multiple fires in
the same calendar minute.
</objective>

<must_haves>
  <truths>
    <truth>CronBus.Fire(slug, when) is non-blocking. A single background goroutine handles the 1s grouping window.</truth>
    <truth>"Already completed" check: storage.LastEntry(dataDir, slug) → if status=="completed" AND scheduled_for equals the fire time (formatted in q.Location), no-op silently.</truth>
    <truth>Single fire + no active session + cycle not completed → flow.StartQuestionnaire(slug, fireTime).</truth>
    <truth>Multiple fires in the same calendar minute → one picker message with `📋 Multiple questionnaires are due. Which would you like to start?` and inline keyboard `[{Text: name, CallbackData: "start:<slug>:<unix>"} ...]`.</truth>
    <truth>If a session is active when fire arrives → silently skip the new fire(s); they will be available via /pull later.</truth>
  </truths>
</must_haves>

  <action>
    Create `internal/commands/cron.go` with:
    - `type PickerSender interface { SendPicker(text string, options []PickerOption) error }`
    - `type PickerOption struct { Label, CallbackData string }`
    - `type CronBus struct { Flow *handler.QuestionFlow; PickerSender PickerSender; Clock func() time.Time; Window time.Duration }`
    - Internal fire-event channel `chan fireEvent{slug string, when time.Time}` with buffer 16.
    - `NewCronBus(flow, picker, clock) *CronBus` — defaults Window=1*time.Second, Clock=time.Now.
    - `Run(ctx)` — loop receiving fires. On first fire start a timer for `Window`. Buffer additional fires that arrive before the timer expires. On timer fire OR ctx.Done → flush.
    - `Fire(slug string, when time.Time)` — non-blocking send to channel (use select-default to drop if buffer full, with a log line).
    - flush(events []fireEvent):
      - Filter out events for sessions that are already active.
      - Filter out events whose cycle is already completed (LastEntry status=="completed" && scheduled_for == when formatted in q.Location).
      - If 0 remain → no-op.
      - If 1 → flow.StartQuestionnaire(slug, when).
      - If ≥2 → picker.SendPicker(msg, options). One PickerOption per slug.
  </action>
  <acceptance_criteria>
    - `internal/commands/cron.go` exists with CronBus, PickerOption, PickerSender, NewCronBus, Run, Fire
    - Fire is non-blocking (send-or-drop on a buffered channel)
    - flush filters out already-completed and already-active fires before deciding single/multi action
    - `go build ./internal/commands/...` exits 0
  </acceptance_criteria>
</task>
