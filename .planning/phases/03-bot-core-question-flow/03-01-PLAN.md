Stand up `internal/bot.Bot` — a wrapper around `tgbotapi.BotAPI` that runs the long-polling
update loop, applies the silent auth gate, and dispatches authorised updates to a pluggable
dispatcher (implemented in 03-02/03-03). No question logic in this plan — just polling,
auth, and routing.
</objective>

  <action>
    Create `internal/bot/bot.go`:
    - `type Dispatcher interface { Handle(ctx context.Context, b *Bot, update tgbotapi.Update) }`
    - `type Bot struct { API *tgbotapi.BotAPI; ChatID int64; dispatcher Dispatcher }`
    - `func New(token string, chatID int64, dispatcher Dispatcher) (*Bot, error)` — `tgbotapi.NewBotAPI(token)`, return wrapped error on failure.
    - `func (b *Bot) Run(ctx context.Context)`:
      ```
      u := tgbotapi.NewUpdate(0); u.Timeout = 30
      updates := b.API.GetUpdatesChan(u)
      defer b.API.StopReceivingUpdates()
      for {
        select {
        case <-ctx.Done(): return
        case update, ok := <-updates:
          if !ok { return }
          if !IsAuthorised(update, b.ChatID) { continue }
          b.dispatcher.Handle(ctx, b, update)
        }
      }
      ```
    - `func (b *Bot) Send(text string) error` — `tgbotapi.NewMessage(b.ChatID, text)`, send via `b.API.Send(msg)`. Returns error.
    - `func (b *Bot) SendMarkdown(text string) error` — same but `msg.ParseMode = tgbotapi.ModeMarkdown`. Used for italic `_Example:_` lines.
    No question logic — that's 03-02.
  </action>
  <acceptance_criteria>
    - `internal/bot/bot.go` exports `Bot`, `Dispatcher`, `New(token, chatID, dispatcher) (*Bot, error)`, `Run(ctx context.Context)`, `Send(text string) error`, `SendMarkdown(text string) error`
    - `Run` returns within 100ms of ctx cancellation when no updates are arriving (the select statement covers `<-ctx.Done()`)
    - Polling configuration uses `Timeout: 30`
    - Unauthorised updates do not invoke `dispatcher.Handle`
    - `go build ./internal/bot/...` and `go vet ./internal/bot/...` exit 0
  </acceptance_criteria>
