<objective>
1. On startup, restore any `session.yaml` files into the session.Manager. If a restored
   session is already fully answered (`CurrentQuestionIndex == len(questions)`),
   finalise it immediately (BOT-09).
2. Implement the dispatcher that the bot.Run loop calls per authorised update:
   - Slash commands route to handler stubs (`/pull`, `/status`, `/list` return
     "Phase 4 will implement this" for now; `/start` returns the help message).
   - Free text with an active session → forward to QuestionFlow.HandleAnswer.
   - Free text with no active session → send the help message (BOT-10).
3. Wire everything in `cmd/bot/main.go` so the binary boots end-to-end.
</objective>

<must_haves>
  <truths>
    <truth>Restore walks every loaded questionnaire's slug, calls `sessions.LoadFromDisk(slug)`, and for any non-nil result either logs `Restored session: <slug> (q=<i>/<n>)` or finalises and logs `Finalised orphan session: <slug>`.</truth>
    <truth>Dispatcher.Handle treats `update.Message.IsCommand() == true` as a slash command. Otherwise it's free text.</truth>
    <truth>If there is exactly one active session, free text answers that session. If zero, fallback help. (Phase 4 will handle the multi-active case via a picker.)</truth>
    <truth>Slash commands: `/start`, `/help` → help text. `/pull`, `/status`, `/list` → reply "Phase 4 will implement this command." Any other slash command → reply "Unknown command."</truth>
    <truth>cmd/bot/main.go starts the bot polling goroutine and the scheduler in parallel, both cancellable via the SIGINT/SIGTERM context.</truth>
  </truths>
</must_haves>

<task id="03-03-T02">
  <title>Dispatcher</title>
  <read_first>
    - internal/bot/bot.go (Dispatcher interface)
    - internal/handler/flow.go
    - .planning/REQUIREMENTS.md (BOT-10)
  </read_first>
  <action>
    Create `internal/handler/dispatcher.go`:
    - `const HelpText = "Send /pull to start a questionnaire now, /status for state, or /list to see schedules."`
    - `type Dispatcher struct { Flow *QuestionFlow }`
    - `func NewDispatcher(flow *QuestionFlow) *Dispatcher`.
    - `func (d *Dispatcher) Handle(ctx context.Context, b *bot.Bot, update tgbotapi.Update)`:
      1. If update.Message == nil → return.
      2. text := update.Message.Text.
      3. If update.Message.IsCommand():
         - command := update.Message.Command()  // strips leading "/"
         - switch command {
             case "start", "help": b.Send(HelpText)
             case "pull", "status", "list": b.Send("Phase 4 will implement /" + command + ".")
             default: b.Send("Unknown command.")
           }
         - return
      4. Free text. Find the single active session: iterate flow.Sessions.Get(slug) for each loaded questionnaire — count non-nil; if exactly one, call flow.HandleAnswer(slug, text). If zero, b.Send(HelpText). If more than one (Phase 4 territory), b.Send(HelpText) for now plus a note: "Multiple active sessions — use /pull to choose."
    Imports: telegram-bot-api/v5, our bot package, log.
  </action>
  <acceptance_criteria>
    - `internal/handler/dispatcher.go` exports `Dispatcher` and `NewDispatcher`
    - `Handle` ignores updates whose Message is nil
    - Slash commands route to b.Send with the prescribed strings
    - Free text with zero active sessions → HelpText sent
    - Free text with one active session → flow.HandleAnswer invoked (verify via a test fake)
    - `go build ./internal/handler/...` exits 0
  </acceptance_criteria>
</task>
