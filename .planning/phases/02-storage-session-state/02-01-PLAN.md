<objective>
Implement `internal/storage` with helpers that **prepend** completed and skipped entries to
`data/<slug>/answers.yaml`. The file must never be rewritten from scratch — new entries go on
top, preserving newest-first order. Writes are atomic via temp-file + rename. Provide a
`LastEntry(dataDir, slug)` helper for Phase 4's past-due algorithm.
</objective>

<must_haves>
  <truths>
    <truth>Atomic write: marshal new entry, read existing file (or empty if missing), write `<new>\n<existing>` to `answers.yaml.tmp` in the same directory, fsync, then `os.Rename` to `answers.yaml`. Same-FS rename is atomic on POSIX.</truth>
    <truth>Completed entry shape (PRD §4.2): keys in order `status: completed`, `scheduled_for`, `completed_at`, `answers: [...]`. Timestamps RFC3339 with the questionnaire's tz offset.</truth>
    <truth>Skipped entry shape: `status: skipped`, `scheduled_for`, `skipped_at`. No `answers` field.</truth>
    <truth>Calling PrependCompleted 100 times on the same file yields 100 entries in reverse-chronological order (newest first); none are lost.</truth>
    <truth>LastEntry returns the FIRST entry from answers.yaml (i.e. the newest) decoded into a typed struct. Returns (nil, nil) when the file does not exist.</truth>
  </truths>
</must_haves>

<task id="02-01-T02">
  <title>Prepend writer + LastEntry</title>
  <read_first>
    - internal/storage/types.go (just created)
    - .planning/REQUIREMENTS.md (STOR-01..03)
    - original-prd.md (§4.2)
  </read_first>
  <action>
    Create `internal/storage/storage.go` exposing:
    - `func PrependCompleted(dataDir, slug string, scheduled, completed time.Time, loc *time.Location, answers []AnswerPair) error`
      Constructs an Entry with Status="completed", ScheduledFor=scheduled.In(loc).Format(RFC3339), CompletedAt=completed.In(loc).Format(RFC3339), Answers=answers. Calls `prepend(...)`.
    - `func PrependSkipped(dataDir, slug string, scheduled, skipped time.Time, loc *time.Location) error`
      Same pattern with Status="skipped".
    - `func LastEntry(dataDir, slug string) (*Entry, error)` — read `<dataDir>/<slug>/answers.yaml`; if missing return (nil, nil); otherwise yaml.Unmarshal into `[]Entry` and return the first (or nil if list is empty).
    - Internal `prepend(path string, e Entry) error`:
      1. `data, err := yaml.Marshal([]Entry{e})` — this yields `- status: ...\n  scheduled_for: ...\n` lines.
      2. Read existing file bytes; if it does not exist, treat as empty.
      3. `os.MkdirAll(filepath.Dir(path), 0o755)`.
      4. Open `<path>.tmp` with O_CREATE|O_WRONLY|O_TRUNC, write `data` then existing bytes, fsync, close.
      5. `os.Rename(tmp, path)`.
    Concurrency: prepend itself is not goroutine-safe across writers (caller is the single session-finalizer). Document this in a comment.
  </action>
  <acceptance_criteria>
    - Exports `PrependCompleted`, `PrependSkipped`, `LastEntry` with the signatures above
    - On a fresh directory, PrependCompleted creates `<dataDir>/<slug>/answers.yaml` containing one entry matching PRD §4.2
    - Calling PrependCompleted then PrependSkipped on the same slug yields a file where the FIRST entry (top of file) is the skipped one, and the SECOND entry is the completed one (newest first)
    - 100 sequential PrependCompleted calls produce 100 entries; `yaml.Unmarshal` into `[]Entry` decodes a slice of length 100
    - LastEntry on a missing file returns `(nil, nil)`
    - LastEntry on a file with two entries returns the entry at index 0 (newest)
    - No call ever truncates existing content without re-emitting it
    - `go vet ./internal/storage/...` exits 0
  </acceptance_criteria>
</task>
