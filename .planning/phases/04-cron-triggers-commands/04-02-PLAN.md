<objective>
Implement `/pull` — for each questionnaire: walk cron ticks between `LastEntry.scheduled_for`
and `now`, prepend `skipped` entries (CMD-02), compute the next-upcoming pending cron
(CMD-03), and either show a picker (CMD-01), reply "active session" (CMD-04), or reply
"all up to date" (CMD-05). Also handle the inline-keyboard callback (`start:<slug>:<unix>`)
to start the selected session.
</objective>

<must_haves>
  <truths>
    <truth>Past-due cap: do NOT prepend more than 365 skipped entries per questionnaire in a single /pull invocation. Cap on the cron-tick iteration prevents pathological loops on fresh files where the baseline lookback is 1 year.</truth>
    <truth>Active-session check uses session.Manager — if ANY slug has an active session, return CMD-04 reply.</truth>
    <truth>Next-upcoming cron is computed via `cron.NewParser` → `.Parse(schedule).Next(now)` after past-due skips have been prepended.</truth>
    <truth>Callback data parser: `start:<slug>:<unixSec>` → starts session at the embedded time. Reject malformed payloads with a `❌ Invalid selection.` reply.</truth>
  </truths>
</must_haves>

  <read_first>

    - .planning/phases/04-cron-triggers-commands/04-CONTEXT.md

  <action>
    Create `internal/commands/parse.go` with:
    - `func NextTrigger(q *loader.Questionnaire, after time.Time) (time.Time, error)` — parses schedule with the 5-field parser, returns the first tick strictly after `after` in q.Location.
    - `func ApplyPastDueSkips(dataDir string, q *loader.Questionnaire, now time.Time, clock func() time.Time) (int, error)`:
      1. last, _ := storage.LastEntry(dataDir, q.Slug)
      2. baseline := if last != nil { parse last.ScheduledFor } else { now.Add(-365*24*Hour) }
      3. sched := parsed cron.Schedule from q.Schedule (in q.Location).
      4. tick := sched.Next(baseline). Loop while tick.Before(now) AND count < 365:
         - Skip the tick that equals baseline (already accounted).
         - storage.PrependSkipped(dataDir, q.Slug, tick, clock(), q.Location).
         - tick = sched.Next(tick); count++
      5. Return count.
    Document the 365 cap in a comment.
  </action>
  <acceptance_criteria>
    - `parse.go` exports NextTrigger and ApplyPastDueSkips
    - For a questionnaire `0 9 * * *` Asia/Kolkata, with baseline=2026-05-15T09:00:00+05:30 and now=2026-05-18T11:00:00+05:30, ApplyPastDueSkips prepends 3 skips (16th, 17th, 18th 9:00)
    - When LastEntry is nil and baseline-fallback would yield more than 365 ticks, the call caps at 365 and returns 365
    - `go vet ./internal/commands/...` exits 0
  </acceptance_criteria>
</task>

<task id="04-02-T02">
  <title>Pull command handler</title>
  <read_first>
    - internal/commands/parse.go (just created)
    - internal/commands/cron.go (PickerSender)
    - internal/session/manager.go
    - .planning/REQUIREMENTS.md (CMD-01..05)
  </read_first>
  <action>
    Create `internal/commands/pull.go` with:
    - `type Pull struct { Flow *handler.QuestionFlow; PickerSender PickerSender; Clock func() time.Time }`
    - `func NewPull(flow, picker, clock) *Pull` — defaults Clock = time.Now.
    - `func (p *Pull) Handle(sender bot.Sender) error`:
      1. If ANY slug has an active session via p.Flow.Sessions.Get → sender.Send(`⚠️ You have an active session in progress. Please finish it first.`); return.
      2. For each slug in p.Flow.Questionnaires, run ApplyPastDueSkips(...). Collect (slug, nextTrigger) pairs where the cycle isn't already completed (LastEntry.ScheduledFor != nextTrigger).
      3. If empty → sender.Send(`✅ All questionnaires are up to date. Nothing to answer right now.`); return.
      4. Otherwise build PickerOption list (Label=q.Name, CallbackData=`start:<slug>:<next.Unix()>`) and call p.PickerSender.SendPicker(text, options) with text=`📋 Pick a questionnaire to start:`.
    - `func (p *Pull) HandleCallback(sender bot.Sender, data string) error` — parse `start:<slug>:<unix>`. Convert unix to time.Time. Reject malformed → sender.Send("❌ Invalid selection."). Otherwise call p.Flow.StartQuestionnaire(slug, scheduled).
  </action>
  <acceptance_criteria>
    - `pull.go` exists with Pull, NewPull, Handle, HandleCallback
    - With an active session: sender receives the active-session reply
    - With nothing pending: sender receives the up-to-date reply
    - With one or more pending: picker.SendPicker called once with N options
    - HandleCallback("start:daily:1747545600") starts the daily session with scheduled = unix 1747545600 (or whatever)
    - HandleCallback("start:bad") replies "❌ Invalid selection."
    - `go vet ./internal/commands/...` exits 0
  </acceptance_criteria>
</task>
