<objective>
Implement `internal/loader` to scan `${DATA_DIR}/*/questionnaire.yaml`, parse each file via
`gopkg.in/yaml.v3`, validate the schema (name / schedule / timezone / questions), and
return a typed slice. Any validation failure must produce a `FATAL: data/<name>/questionnaire.yaml: <reason>`
error suitable for main.go to print and exit on. On success, emit
`Loaded N questionnaire(s): [name-1, name-2, ...]` to stdout.
</objective>

<must_haves>
  <truths>
    <truth>Loader.Load(dataDir) walks only one directory level deep — `${DATA_DIR}/<slug>/questionnaire.yaml`. Nested subdirectories below `<slug>/` are ignored.</truth>
    <truth>Each validation error is wrapped to read `data/<slug>/questionnaire.yaml: <reason>` — the loader does not prefix "FATAL:"; main.go does that when it exits.</truth>
    <truth>Schedule must validate via robfig/cron `cron.ParseStandard` (5-field). 6-field expressions are rejected.</truth>
    <truth>Timezone must validate via `time.LoadLocation` — the resulting *time.Location is stored on the parsed struct for use by the scheduler.</truth>
    <truth>Questions must be a non-empty array; each item's `question` must be a non-empty string; `example` is optional.</truth>
    <truth>The directory slug (e.g. `daily-standup`) is captured as the questionnaire's filesystem ID — distinct from the display `name` field.</truth>
  </truths>
</must_haves>

<task id="01-02-T02">
  <title>Implement Load(dataDir) — scan, parse, validate</title>
  <read_first>
    - internal/loader/types.go (just created)
    - original-prd.md (§4.1 schema; §11 US-001 acceptance criteria)
    - .planning/REQUIREMENTS.md (LOAD-01..04)
  </read_first>
  <action>
    Create `internal/loader/loader.go` exposing `func Load(dataDir string) ([]*Questionnaire, error)`:
    1. `os.ReadDir(dataDir)`; iterate entries that are directories.
    2. For each subdir `slug`, attempt to read `<dataDir>/<slug>/questionnaire.yaml`.
       Missing file → skip silently (subdirs without a questionnaire are allowed for future use).
    3. `yaml.Unmarshal` into Questionnaire. Marshalling error → return `&LoadError{Path: "data/"+slug+"/questionnaire.yaml", Reason: err.Error()}`.
    4. Validation, returning a LoadError on first failure (do not collect; fail fast):
       - `Name` non-empty (trimmed).
       - `Schedule` parses via `cron.ParseStandard` — wrap the parse error: "invalid schedule: <err>".
       - `Timezone` non-empty; `time.LoadLocation(tz)` must succeed — wrap as "invalid timezone: <err>". Store the *Location on the struct.
       - `Questions` length >= 1; each item's `Question` field non-empty after trim. Wrap as "questions[N]: <reason>".
    5. Set `q.Slug = slug` on success.
    6. Return collected slice sorted by `slug` (lexicographic) for deterministic logging.
    7. After all parsing succeeds, log `Loaded N questionnaire(s): [slug1, slug2, ...]` via `log.Printf` to stdout (single line, slugs comma-space-separated inside square brackets).
    8. If dataDir contains zero questionnaires, return `(nil, &LoadError{Path: dataDir, Reason: "no questionnaires found"})`.
  </action>
  <acceptance_criteria>
    - `internal/loader/loader.go` exports `func Load(dataDir string) ([]*Questionnaire, error)`
    - Loading a directory with one valid questionnaire returns a slice of length 1 with Slug populated to the directory name
    - Loading a directory where one file has `schedule: "invalid"` returns an error whose .Error() starts with "data/<slug>/questionnaire.yaml: invalid schedule:"
    - Loading a directory where one file has `timezone: "Mars/Olympus"` returns an error whose .Error() starts with "data/<slug>/questionnaire.yaml: invalid timezone:"
    - Loading a directory where one file has `questions: []` returns an error mentioning "questions"
    - Loading an empty directory returns an error mentioning "no questionnaires found"
    - On successful load, stdout contains exactly one line matching `^Loaded \d+ questionnaire\(s\): \[.*\]$`
    - Loaded questionnaires are sorted by Slug ascending
    - `go build ./internal/loader/...` and `go vet ./internal/loader/...` exit 0
  </acceptance_criteria>
</task>
