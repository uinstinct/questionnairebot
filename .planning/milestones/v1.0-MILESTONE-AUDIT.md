## Cross-Phase Integration

Inline check of `cmd/bot/main.go` (integration checker agent not installed). Wiring graph:

```
config.Load
   └─► loader.Load(DataDir)             ─► []Questionnaire
         └─► session.NewManager(DataDir) ─► sessions
               └─► handler.New(_, sessions, DataDir, Qs) ─► flow
                     └─► handler.NewDispatcher(flow) ─► disp
                           └─► bot.New(token, chatID, disp) ─► b
                                 └─► flow.Sender = b
                                 └─► handler.Restore(flow)
                                 └─► commands.NewCronBus(flow, b, time.Now) ─► bus
                                 └─► commands.NewPull/Status/List ─► adapter ─► disp.Attach
                                 └─► scheduler.New(Qs, bus.Fire) ─► sched.Start
```

All 8 internal packages from phases 1–4 are wired into `cmd/bot/main.go`. Phase 5 adds `internal/e2e` (integration tests) and Docker packaging — neither alters the runtime graph.

**Cross-phase contracts verified:**
- Phase 1 → 4: `scheduler.Start` callback invokes `bus.Fire(slug, ts)` (cmd/bot/main.go:58).
- Phase 2 → 3: `handler.New` receives `session.Manager` and persists via `storage` after each answer.
- Phase 3 → 4: `disp.Attach(commands.NewAdapter(...))` connects `/pull`, `/status`, `/list` to the dispatcher.
- Phase 4 → 3: `bus` receives `flow` so cron fires can start sessions in the same `QuestionFlow`.
- Phase 5 → all: `internal/e2e/helpers_test.go` boots the identical component graph as `main.go`, exercising the wiring end-to-end.

No wiring gaps detected.
