<objective>
Add two E2E tests that drive the bot end-to-end against a real Telegram test bot using
`TEST_TELEGRAM_BOT_TOKEN` and `TEST_TELEGRAM_CHAT_ID` from the environment: (a) cron
fire → first question → user replies through to completion → `answers.yaml` contains the
expected `completed` entry; (b) two pending questionnaires → `/pull` → picker shown →
user taps one → completes → second remains pending in the next `/pull`. Document the
required environment variables in a `## Running Tests` section in `README.md`. All
tests run via `go test ./... -tags integration`.

`autonomous: false` — these tests require live Telegram credentials. The executor must
either (i) source them from the user's environment if present or (ii) flag the plan for
manual run with credentials supplied.
</objective>

<task id="05-03-T03">
  <title>E2E dual-pending /pull picker test (TEST-04)</title>
  <read_first>
    - internal/e2e/helpers_test.go
    - internal/commands/pull.go
    - internal/commands/cron.go
  </read_first>
  <action>
    Create `internal/e2e/pull_picker_test.go` with `//go:build integration` tag.

    Test `TestE2EPullPickerWithTwoPending`:
    1. `requireTestEnv`, tempDir with two questionnaires `qa` and `qb`, each with 1 question.
    2. Boot the rig.
    3. Fire both cron events back-to-back within the bus window: `rig.bus.Fire("qa", now); rig.bus.Fire("qb", now)`.
    4. `probe.WaitForMessage(t, 10s, contains("Multiple questionnaires are due"))` — the auto-trigger picker.
    5. Dismiss it (this test exercises `/pull` specifically). Either tap one button to clear or simply ignore and proceed.
    6. After auto-trigger picker resolved (or skipped), drive `/pull`: `probe.Send("/pull")`.
    7. `probe.WaitForMessage(t, 10s, isInlineKeyboardWithButtons("qa", "qb"))` — assert picker contains two buttons.
    8. `probe.SendCallback("start:qa:<unix>")` (build the callback data from the captured picker payload).
    9. `probe.WaitForMessage(t, 10s, contains("qa question text"))`.
    10. `probe.Send("answer")` → wait for completion message.
    11. `probe.Send("/pull")` again → assert picker now contains only `qb`.

    Acknowledge that step 4 may race depending on which arrives first (the cron picker vs `/pull`). Use the probe message log to identify and assert on the correct frames.
  </action>
  <acceptance_criteria>
    - File `internal/e2e/pull_picker_test.go` exists with `//go:build integration` tag
    - Contains `func TestE2EPullPickerWithTwoPending(t *testing.T)`
    - With env vars set, test asserts: (a) picker initially shows both questionnaires, (b) after completing one, the next `/pull` shows only the remaining one
    - With env vars unset, test is skipped (not failed)
  </acceptance_criteria>
</task>
