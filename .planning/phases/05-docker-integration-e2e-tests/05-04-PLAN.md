# Plan 05-04 — E2E User-Action Mirroring to Telegram

## Goal

Every message the test sends *on behalf of the user* should also appear in the
real Telegram chat so that a test run produces a readable two-way conversation,
not just a stream of bot replies.

---

## Background

After the 05-03 probe redesign, the E2E infrastructure works as follows:

| Direction | Current behaviour |
|-----------|------------------|
| Bot → User | `e2eSender` sends via real Telegram **and** records to channel |
| User → Bot | `botRig.inject` dispatches directly into the dispatcher — **not sent to Telegram** |

An observer watching the Telegram chat during a test run sees only the bot side.

---

## Approach

Before `botRig.inject` dispatches a user message in-process, send a
**mirror message** to the Telegram chat using the real bot API so both sides of
the conversation are visible.

The mirror is sent as a distinctly-formatted bot message to make it clear it
is a test simulation, not a real user reply.

**Proposed format:** `👤 <text>`  
Example: the test sends `"fine"` → Telegram shows `👤 fine` then the bot replies.

---

## Changes Required

### 1. `internal/e2e/helpers_test.go` — add `logUserAction` to `e2eSender`

Add a method that sends the mirror message to Telegram without recording it to
the `out` channel (it is not a bot reply, so the probe must not wait for it):

```go
func (s *e2eSender) logUserAction(text string) {
    _ = s.real.Send("👤 " + text)
}
```

### 2. `internal/e2e/helpers_test.go` — call `logUserAction` inside `botRig.inject`

```go
func (r *botRig) inject(text string) {
    r.sender.logUserAction(text)   // ← new line
    // ... existing update construction and disp.Handle call unchanged
}
```

No changes needed to `probeClient.send` or `probeClient.sendCallback` — they
both already delegate to `inject`.

## Expected Telegram Chat Transcript (happy path)

```
👤 (cron fires — no message from user)
Bot: How was today?
👤 fine
Bot: Anything else?
👤 nope
Bot: ✅ Daily complete!
```

```
👤 /pull
Bot: [inline keyboard: QA | QB]
👤 (cron fires)
Bot: QA question?
👤 answer-qa
Bot: ✅ QA complete!
👤 /pull
Bot: [inline keyboard: QB]
```

---

## Out of Scope

- Sending messages as a real Telegram user account (requires MTProto / user
  session — significant extra infrastructure, not warranted here)
- Recording mirror messages in `probeClient.log`
- Changing the format of the mirror prefix (`👤`) — trivial to update later
