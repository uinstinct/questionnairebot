<objective>
Add four `integration`-tagged tests covering: (a) partial `session.yaml` resume on restart,
(b) past-due cron cycle producing a skip entry then surfacing the next upcoming cron via
`/pull`, (c) `/status` output structure across multiple questionnaires, and (d) malformed
`questionnaire.yaml` causing fatal exit. All tests are hermetic (temp DATA_DIR), use the
real on-disk YAML layer, and run via `go test ./... -tags integration`.
</objective>

<must_haves>
  <truths>
    <truth>Every new test file has `//go:build integration` as its first line, so the default `go test ./...` run does not pick them up.</truth>
    <truth>Each test creates a fresh temp directory via `t.TempDir()`, writes the required `questionnaire.yaml` / `session.yaml` / `answers.yaml` fixtures inline, and asserts via `testify`.</truth>
    <truth>TEST-05 (session resume): write a partial `session.yaml` with `current_question_index=1`, call `handler.Restore`, then drive `handler.OnText` with any text; assert the bot sends the question at index 1, not index 0.</truth>
    <truth>TEST-06 (past-due skip): seed `answers.yaml` with a baseline completion older than the current cron schedule, invoke `commands.Pull` at a "now" beyond multiple cron ticks; assert `skipped` entries are prepended for each missed tick and the picker reflects only the next upcoming cron per questionnaire.</truth>
    <truth>TEST-07 (`/status`): seed two questionnaires with different states (one with a completed entry, one never answered, one with an active session); assert the `/status` output contains the expected name, last-answered timestamp / `Never`, next scheduled time, and state label (`✅ Done` / `🔄 In Progress` / `⏳ Pending`).</truth>
    <truth>TEST-08 (malformed YAML): write an invalid `questionnaire.yaml` (e.g., missing `schedule`), call `loader.Load`, assert it returns a non-nil error whose message starts with `FATAL:` or matches the existing loader error format used by `cmd/bot/main.go fatal()`.</truth>
    <truth>`go test ./... -tags integration` exits 0 on a clean checkout (and exits 0 without the tag because the build constraint excludes the files).</truth>
  </truths>
</must_haves>
