<objective>
Initialize the Go module, scaffold `cmd/bot/main.go` with a clean startup/shutdown skeleton,
and implement `internal/config` to load `.env` via `godotenv` and validate
`TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`, and `DATA_DIR` before any network call. Missing or
invalid env vars must cause `os.Exit(1)` with a descriptive `FATAL: ...` stderr message.
</objective>

<must_haves>
  <truths>
    <truth>Binary must exit with non-zero status BEFORE any network call when TELEGRAM_BOT_TOKEN or TELEGRAM_CHAT_ID is missing or empty.</truth>
    <truth>TELEGRAM_CHAT_ID must parse as int64 (Telegram chat IDs can be negative for groups); reject if not a valid integer.</truth>
    <truth>DATA_DIR defaults to "./data" when unset, but the directory must exist at startup (fatal otherwise).</truth>
    <truth>Error messages must be prefixed with "FATAL: " and printed to stderr.</truth>
    <truth>config.Load() returns a typed Config struct and a structured error; main.go is the only place that calls os.Exit.</truth>
  </truths>
</must_haves>

<task id="01-01-T02">
  <title>Implement internal/config — env loading + validation</title>
  <read_first>
    - .planning/phases/01-foundation-config-loader-scheduler-init/01-CONTEXT.md
    - original-prd.md (§7 FR-2; §11 US-001)
    - .planning/REQUIREMENTS.md (CFG-01)
  </read_first>
  <action>
    Create `internal/config/config.go` exposing:
    - `type Config struct { BotToken string; ChatID int64; DataDir string }`
    - `func Load() (*Config, error)` — calls `godotenv.Load()` (ignoring fs.ErrNotExist;
      env-only deployments must still work), reads `TELEGRAM_BOT_TOKEN`,
      `TELEGRAM_CHAT_ID`, `DATA_DIR` via `os.Getenv`, returns a structured `*ConfigError`
      naming the missing/invalid key.
    Validation rules:
    - BotToken: non-empty string; trim whitespace.
    - ChatID: must parse via `strconv.ParseInt(s, 10, 64)`. Empty or invalid → error.
    - DataDir: default to `./data` if unset; `os.Stat` must show it is a directory.
    No network calls anywhere in this package. Errors are values, not panics.
  </action>
  <acceptance_criteria>
    - `internal/config/config.go` defines `Config` struct with exported BotToken (string), ChatID (int64), DataDir (string)
    - `Load()` returns non-nil *Config and nil error when all three vars are set and DataDir is a real directory
    - `Load()` returns nil *Config and error whose message contains the missing variable name (e.g. "TELEGRAM_BOT_TOKEN") when that variable is missing or empty
    - `Load()` returns error containing "TELEGRAM_CHAT_ID" when ChatID is non-numeric (e.g. "abc")
    - `Load()` returns error mentioning "DATA_DIR" when DataDir is set to a path that does not exist
    - `go vet ./internal/config/...` exits 0
  </acceptance_criteria>
</task>

    - SIGTERM/SIGINT cause the binary to exit 0 within 2 seconds
    - `go build ./cmd/bot/...` exits 0
